## Interpretation

**Permitted conclusion:** Within this authored 12-case workflow model, the assisted arm uses fewer modeled base active-review seconds and human checkpoints while both arms match the developer policy labels.

The equal mismatch counts do not establish error reduction. Because both arms and labels are developer-authored and the set is tuned/synthetic, they establish only self-consistency on this fixed policy fixture.

**Do not extrapolate:** ROI; labor savings; headcount reduction; production throughput; production error reduction; human accuracy improvement; causal impact; population generalization; statistical significance.

Key limitations:

- The task list is a workflow model, not a time-and-motion observation.
- Task weights are authored assumptions chosen before derived-result generation and are visible for replacement by future measured values.
- Every task represents active human review; system latency and downstream execution are excluded.
- The manual arm is a reasonable standardized checklist and is not intentionally deprived of history, duplicate, or variance checks.
- Both actions and labels are developer-authored, so mismatch counts are a policy-regression signal rather than an estimate of human or production error.
- The 12 cases are synthetic and developer-selected, not random, representative, independent, or production sampled.

## Reproduce

From the repository root:

    npm run impact:check

To run only the 12-case agent-to-raw action/trace replay comparison:

    npm run impact:check-raw

To regenerate derived outputs only after the committed raw/source identity is already valid:

    npm run impact:write

After any replay source or protocol change, first commit the complete clean source as commit A, then run:

    npm run impact:refresh-source

Review and commit only the refreshed raw/results as the evidence-only descendant commit B. The refresh command refuses a dirty or uncommitted commit A.

The check revalidates the fixed denominator, exact projection from eval/dataset.ts, task catalogs, raw replay shape, policy boundaries, input hashes, and byte-for-byte generated outputs.
